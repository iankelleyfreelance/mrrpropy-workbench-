What this site covers

This documentation deliberately focuses on the stable public API and the repository workflow around the scientific algorithms. The RaProMPro kernels remain reference scientific code; optimizations are handled in controlled wrapper paths and equivalence tests.

Entry points

  • mrrpropy.raw_class.MRRProData is the main API for loading, processing, plotting and analysis.
  • mrrpropy.cli.main exposes the lightweight CLI entry point.
  • mrrpropy.hexagram and mrrpropy.utils contain analysis helpers used by the high-level API.

Repository workflow

Tests use bundled NetCDF fixtures under tests/data/. Generated figures go to tests/figures/, and generated non-figure test outputs go to tests/generated/.

Typing scope

Typing is enforced first on the public API and workflow utilities. The scientific kernels are intentionally treated more conservatively to avoid accidental behavioural change.
