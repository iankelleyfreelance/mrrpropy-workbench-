Typing strategy

Current scope

The repository is marked as typed and includes a baseline mypy configuration focused on:

  • the public package entry points,
  • the high-level MRRProData API,
  • workflow scripts, CLI and test fixtures.

Why not full strict typing everywhere

The RaProMPro scientific kernels are intentionally treated more conservatively. Over-aggressive refactors in those files increase the risk of changing scientific behaviour while chasing type cleanliness.

Local check

uv run mypy

Recommended next steps

  • Expand annotations around plotting and analysis methods in raw_class.py.
  • Add small typed helper aliases for common dataset/array shapes.
  • Keep scientific kernels behind equivalence tests before tightening types there.
